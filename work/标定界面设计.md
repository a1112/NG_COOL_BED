·  标定界面与 CameraView 设计类似

    目录结构安装camera_view的类似划分方法，不同区域不同文件夹
    
    建立 calibration_view/
        创建 CalibrationView.qml
    添加到 MainViewLayout.qml
    
    CalibrationView 功能：
        显示标定图像，显示多边形标签，标签可调整，显示透视结果图像

    布局 左
            上： 
                左：切换标注文件夹的下拉框，默认显示当前的标注文件夹
                右： 添加，删除 使用 按钮
                    添加：点击后，弹窗添加界面
                        弹窗的添加界面：
                            名称  （用于创建文件夹的名称） 
                            模板  （下拉框，从当前标定的文件夹中选择，默认选中当前使用的）
                            图像来源：（横向单选） （从相机获取（离线模式不可选） ， 拷贝（默认））
                            确认，取消按钮。 名称与现有一致则 弹窗提示
                        添加逻辑： api 接收到标定标定创建请求，在对应的目录创建文件夹，拷贝的话将全部文件拷贝过去，从相机则用相机文件替换图像
            下： 根据 CameraManage.json 进行
                根据不同的 group 来进行显示，
        右   
            上：
                左：
                    根据透视变化组合出来的透视图像
                    图层1：
                        矩形样式的位置标注（来源 config/calibrate/mapping/{对应标注}/{对应拼接名称}.xml） 在添加时，mapping也进行添加
                        矩形标注为可调整
                右：
                    右边为可管理的对象设置目录，按照 xml 中的对象
                    保存按钮：保存到xml 文件
            下：
                上：页眉：切换相机的单选框， 从相机采图 保存采集 按钮（点击后从相机中采集图像），刷新透视 按钮（点击后刷新上面的透视），自动刷新 多选  
                下：  
                    左： 相机图像（标定文件夹）
                    右： 对象管理（目前只有Area）
                        透视方向选择（下拉）
                        透视的目标尺寸（像素）
                        （基本上根据（CameraManage.json）中来）
